one way mode인 경우에 1. long 포지션이 있는데 가지고 있던 양보다 더 많은 양의 sell order가 체결되는 경우 2. short 포지션이 있는데 가지고 있던 양보다 더 많은 양의 buy       
order가 체결되는 경우에 포지션이 바로 뒤집힘

예시:

Long 포지션 (1 BTC 보유)

Sell 0.5 BTC → 0.5 BTC 남음 (롱 유지)
Sell 0.7 BTC → -0.2 BTC (숏으로 전환)


Short 포지션 (-1 BTC 보유)

Buy 0.5 BTC → -0.5 BTC 남음 (숏 유지)
Buy 1.2 BTC → +0.2 BTC (롱으로 전환)

바이비트는 포지션 사이드가 바로 뒤집혀서 들어오고 바이낸스는 pa의 부호가 반전되고 비트겟은 홀드 사이드가 반전돼

발생하는 문제
1. 기존에 열려있던 포지션은 닫히지 않고 새로운 포지션이 열림
2. 하나의 오더가 진입, 청산을 둘 다 담당하게 되지만 디비엔 하나만 저장됨
